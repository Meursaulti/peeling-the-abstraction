# C语言编译过程

写一个 simple.c，然后执行：

```bash
gcc simple.c -o simple
```

整个过程大致可以拆成下面几步：

---

## 1. 预处理（Preprocessing）

先做一轮“纯文本替换”：

- `#include` → 头文件直接展开
- `#define` → 宏替换
- `#ifdef` → 条件编译
- 注释会被删掉

为了避免头文件被重复 include，需要用 Header Guard：

```c
#ifndef COMMON_H_
#define COMMON_H_

// 声明

#endif
```

这里要注意：

> 这一阶段完全不懂 C 语义，就是在做字符串替换

输出是 `.i` 文件。

---

## 2. 编译（Compilation）

这一阶段才真正“看懂代码”：

- 语法 / 语义检查
- 生成中间表示（IR）
- 做一些优化
- 最后变成汇编（`.s`）

一个关键点：

> 函数可以没有实现，只要有声明就能过编译

比如：

```asm
call add
```

这里还是符号，不是地址。

---

## 3. 汇编（Assembly）

把 `.s` 转成机器码，生成 `.o`

同时做三件事：

- 指令转二进制
- 建 `symbol table`
- 建 `relocation table`

`relocation table`：

> 对于全局变量的读取 / 存储，以及外部函数调用进行标记，留到链接阶段补全。

---

## 4. 链接（Linking）

链接器生成可执行文件：

- 把多个 `.o` 拼起来
- 找函数实现
- 把之前没填的地址补上

分两种：

### 静态链接

- 从库直接拷需要的目标文件一起链接
- 文件变大，但运行独立

### 动态链接（默认）

- 只保留库的标识
- 运行时再加载动态库
- 节省磁盘空间
- 库更新通常不需要重新编译程序

# 关键路径（Critical Path）完整流程

同步电路的运行依赖于时钟周期的节奏，数据在寄存器之间传递。

---

## 1. 同步路径概览

```text
        ┌──────────┐
CLK ───▶│ Register A│
        └────┬─────┘
             │
             ▼
      ┌─────────────┐
      │Combinational│
      │   Logic     │
      └────┬────────┘
           │
           ▼
        ┌──────────┐
CLK ───▶│ Register B│
        └──────────┘
```

---

## 2. 时钟上升沿的宏观动作

在每一个时钟上升沿（Rising Edge）触发的瞬间：

- **寄存器 A**：采样并接收当前输入端的新数据 `N`

- **寄存器 B**：采样并接收来自组合逻辑的输出  
  ——这实际上是寄存器 A 在**上一个周期**接收的数据 `N-1` 经过组合逻辑处理后的结果。

> ⚠️ 核心要求：寄存器的输入端必须满足特定的时序约束，否则会陷入亚稳态。  
> 关于寄存器为何需要这些时间，详见：[主锁存器与从锁存器](#主锁存器与从锁存器)

---

## 3. 详细流程演进（以寄存器 A 的输出为起点）

### 1. 内部再生与输出（`t_pcq`）

在上升沿瞬间，寄存器 A 捕获数据。内部电路启动快速的**反馈再生机制**，将具有倾向性的微弱电压：

```text
0.55V → 0.75V → 0.9V
```

迅速放大锁定。

直到内部电压稳定、不再“反复横跳”，并形成明确的 `0/1` 逻辑电平时，输出端 `Q` 才会更新为合法稳定的电平。

这段从时钟上升沿到 `Q` 端输出稳定的时间称为：

```text
t_pcq（Clock-to-Q delay）
```

---

### 2. 组合逻辑处理（`t_comb`）

信号离开寄存器 A 后，进入组合逻辑块。

这本质上是一系列逻辑门（级联、并联）和导线的组合。信号在这里穿梭、运算，消耗的时间称为：

```text
t_comb
```

这通常是整条路径中耗时最长的步骤。

---

### 3. 门前数据建立（`t_setup`）

处理后的信号到达寄存器 B 的门前。

为了让寄存器 B 在第二个上升沿到来时能够得到稳定数据，信号必须提前到达并保持稳定。

这一段“预留时间”就是：

```text
t_setup（建立时间）
```

---

## 4. 关键路径延迟公式

为了保证系统正常工作，一个时钟周期 `T` 必须容纳整条路径的延迟：

```text
T ≥ t_pcq + t_comb + t_setup
```

如果时钟周期小于这个延迟：

- 新数据可能还没流到寄存器 B
- 或者内部电荷还没建立稳定
- 导致不满足建立时间（Setup Violation）
- 最终出现逻辑错误甚至亚稳态

---

# 主锁存器与从锁存器

为什么寄存器接收数据需要“建立期”和“保持期”？

这需要深入到主从 D 触发器（Master-Slave D Flip-Flop）的内部。

---

## 1. 为什么需要建立期（`t_setup`）？

寄存器的核心是：

```text
主锁存器（Master Latch）
        +
从锁存器（Slave Latch）
```

串联组成。

---

### 开门灌水（下降沿开始）

当时钟从高电平降为低电平（下降沿）时：

- 主锁存器的“外门”打开
- 外部电荷开始流入主锁存器

---

### 建立倾向（`t_setup` 窗口）

在下一个上升沿到来之前的这一小段关键时间内：

```text
输入必须保持稳定
```

这段时间是给物理电荷去填充内部节点电容（水槽）的。

只有输入稳定，主锁存器内部才能建立起一个：

```text
具有明确倾向性的电平
```

例如：

```text
0.6V
```

---

## 2. 为什么需要保持期（`t_hold`）？

### 关门瞬间（上升沿来到）

当时钟上升沿来到时：

- 主锁存器的“外门”闭合
- 内部反馈电路瞬间接管

该反馈电路通常由两个反相器组成。

---

### 再生放大

反馈电路利用刚才建立的倾向，执行再生机制：

```text
0.6V → 0.2V → 0.9V
```

迅速拉向标准逻辑电平。

---

### 防止污染（`t_hold`）

由于物理上的传输门无法“瞬间锁死”：

在关门的一刹那，如果外部输入突然改变，杂乱电荷可能会“滑”进主锁存器，污染已经建立好的倾向。

因此：

```text
在上升沿之后的一小段时间内，
输入也必须继续保持稳定
```

这一段时间称为：

```text
t_hold（保持时间）
```

---

## 3. 数据交接与输出

与此同时，上升沿也打开了连接主从锁存器的“内门”。

### 从锁存器动作

此时从锁存器：

- 断开自己的正反馈环路
- 停止维持旧数据

---

### 物理传递

主锁存器中刚刚完成再生的强劲电压信号：

```text
迅速冲入从锁存器
```

强行改变其内部电平。

---

### Q 端稳定

经过极短的物理传播后：

```text
输出端 Q 呈现出合法、稳定的新电平
```

---

## 4. 周期循环

当下一个下降沿来到：

- 主从之间的通路再次关闭
- 主锁存器的大门重新打开
- 准备迎接下一轮数据“灌溉”
